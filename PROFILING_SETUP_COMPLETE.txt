QUICK START EXAMPLES
════════════════════════════════════════════════════════════════════════

1. Basic Profiling (Easiest)
   $ ./profile_gpu.sh -- python your_script.py

2. With Custom Annotations
   # In your Python code:
   from profile_utils import nvtx_range
   
   with nvtx_range("my_operation"):
       output = model(input)
   
   # Then profile:
   $ ./profile_gpu.sh -- python script.py

3. Benchmark Inference
   $ python profile_nsight.py --mode benchmark --model-type simple
   
   # Or with RF-DETR:
   $ nsys profile --trace=cuda,nvtx --output=result \
       uv run python profile_nsight.py --mode benchmark --model-type rfdetr

4. Analyze Existing Profiles
   $ python profile_nsight_analysis.py profile.nsys-rep
   
   # Detailed analysis:
   $ python profile_nsight_analysis.py profile.nsys-rep --full

5. View Results
   $ nsys stats profile.nsys-rep --report cuda_api_sum
   $ nsys-ui profile.nsys-rep  # GUI (requires X11)

Accurate CUDA Timing:
   from profile_utils import CUDATimer
   
   with CUDATimer() as timer:
       output = model(input)
   print(f"Time: {timer.elapsed_time_ms:.2f} ms")

Benchmark with Stats:
   from profile_utils import benchmark_inference
   
   stats = benchmark_inference(model, input, num_iterations=100)
   print(f"Mean: {stats['mean_ms']:.2f} ± {stats['std_ms']:.2f} ms")

WORKFLOW RECOMMENDATIONS
════════════════════════════════════════════════════════════════════════

For Quick Profiling:
  → Use profile_gpu.sh directly (no code changes needed)

For Detailed Analysis:
  → Add NVTX annotations with profile_utils.py
  → Profile with profile_gpu.sh
  → Analyze with profile_nsight_analysis.py

For Automated Benchmarking:
  → Use profile_nsight.py with nsys profile wrapper

For Production Monitoring:
  → Keep NVTX annotations in code (negligible overhead)
  → Enable profiling only when needed

IMPORTANT NOTES
════════════════════════════════════════════════════════════════════════
• nsight-python's annotation.annotate() does NOT support nesting
  → Use torch.cuda.nvtx or profile_utils.nvtx_range() instead

• Profile files are in .gitignore (won't be committed)

• All scripts work with 'uv run' for environment management

• NVTX annotations have negligible overhead when not profiling
